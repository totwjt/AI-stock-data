# 数据同步模块实现计划（Tushare → PostgreSQL）

## 1 项目目标

在现有项目中新增一个 **数据同步模块（data-sync）**，用于将 Tushare API
获取的数据同步到本地 PostgreSQL 数据库。

该模块必须满足以下要求：

-   不影响现有 API 功能
-   模块独立运行
-   支持全量同步与增量同步
-   结构清晰，便于 AI 理解并继续开发
-   数据结构稳定，便于策略系统读取

------------------------------------------------------------------------

当前项目结构示意：

# 3 数据同步模块职责

数据同步模块负责：

    Tushare API
          ↓
    数据获取
          ↓
    数据清洗
          ↓
    写入 PostgreSQL

模块不负责：

    策略生成
    回测
    交易执行

------------------------------------------------------------------------

# 7 同步策略

## 7.1 全量同步

适用于：

    stock_basic
    trade_cal
    历史行情

执行方式：

    python sync_full.py

流程：

    获取 API 数据
    ↓
    批量写入数据库

------------------------------------------------------------------------

## 7.2 增量同步

适用于：

    daily
    daily_basic
    adj_factor

策略：

    只同步最近交易日数据

执行时间：

    每日 16:00 以后

------------------------------------------------------------------------

# 8 同步任务设计

同步任务独立实现。

示例：

    sync_stock_basic.py
    sync_trade_calendar.py
    sync_daily.py
    sync_adj_factor.py

统一入口：

    sync_runner.py

执行方式：

    python -m data_sync.sync_runner

------------------------------------------------------------------------

# 9 数据写入策略

写入方式：

    UPSERT

SQL 示例：

    INSERT INTO table (...)
    VALUES (...)
    ON CONFLICT (...)
    DO UPDATE

目标：

    避免重复
    保证幂等

# 15 Python运行环境

项目使用 **Python 虚拟环境（venv）**。

推荐结构：

    project_root
    │
    ├─ .venv/
    ├─ requirements.txt
    ├─ app/
    ├─ data_sync/

初始化：

    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt

数据同步模块必须在虚拟环境中运行。
